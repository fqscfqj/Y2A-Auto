自定义 FFmpeg 视频编码器参数。仅填写视频编码相关参数，音频参数固定为 AAC 320kbps 双声道。

默认转码参数（按分辨率自动选择）：
  • 4K (2160p)：25Mbps 目标码率，30Mbps 峰值，60M 缓冲
  • 1080p：12Mbps 目标码率，15Mbps 峰值，30M 缓冲
  • 720p：8Mbps 目标码率，10Mbps 峰值，20M 缓冲
  • 音频：AAC 320kbps 双声道
GPU编码说明：
  • Docker用户：需配置GPU穿透，详见 docker-compose.yml 中的注释说明
  • NVIDIA：需安装 NVIDIA 驱动（支持 NVENC）
  • Intel：需安装 Intel Media SDK 或 oneVPL
  • AMD (Windows)：需安装 AMD 驱动及 AMF 组件
  • AMD (Linux)：使用 VAAPI，需安装 mesa-va-drivers

FireRedASR2S 语音识别

高性能本地语音识别服务，支持实时转写与多语言识别

关于 FireRedASR2S

FireRedASR2S 提供完整语音流水线接口，本页面按官方推荐的 /v1/process_all 方式调用。

项目地址：https://github.com/fqscfqj/FireRedASR2S-DockerApi

启用后将优先使用 FireRedASR2S 进行语音识别，而非 Whisper API

系统会自动拼接 /v1/process_all?force_refresh=false

调用方式（固定）

POST {FIREREDASR_BASE_URL}/v1/process_all?force_refresh=false

Header: X-API-Key: <你的 Key>（如果服务开启了鉴权）

支持的模型：tiny, base, small, medium, large, large-v2, large-v3

如：en, zh, ja, ko 等

当前按 /v1/process_all 方式调用。模型/语言/提示词/词级时间戳/temperature/beam_size 不作为请求参数发送，仅保留历史兼容配置。
使用说明
  • 需要先部署 FireRedASR2S Docker 服务
  • 确保服务地址可访问，默认端口为 8000
  • 启用后将替代 Whisper API 进行语音识别
  • 建议使用 large-v3 模型以获得最佳识别效果

VAD 参数

任务并发控制

同时处理的最大任务数量（包括下载、翻译等操作），推荐2个以平衡性能与内存使用

同时上传到AcFun的最大视频数量，建议设置为1

并发控制说明
  • 任务并发：控制整体任务处理的并发数，影响系统资源使用（推荐2个）
  • 上传并发：控制同时上传到AcFun的视频数，建议设置为1
  • 推荐配置：任务并发2，上传并发1（平衡性能与内存使用）
  • 内存优化：系统会在高内存使用时自动降低并发数
